01 · 기본 View Entity 기본

ZI_JNC_EDU_01_FLIGHT

목적

가장 단순한 View Entity의 골격을 익힙니다 — 소스 지정, key, 별칭(as), 투영.

소스

@AbapCatalog.viewEnhancementCategory: [#PROJECTION_LIST]
@AccessControl.authorizationCheck: #NOT_REQUIRED
@EndUserText.label: 'EDU 01 - Basic Flight View'
define view entity ZI_JNC_EDU_01_FLIGHT
  as select from sflight
{
  key carrid    as AirlineID,
  key connid    as ConnectionID,
  key fldate    as FlightDate,
      price     as Price,
      currency  as CurrencyCode,
      planetype as PlaneType,
      seatsmax  as SeatsMax,
      seatsocc  as SeatsOccupied
}

해설

  • @AbapCatalog.viewEnhancementCategory: [#PROJECTION_LIST] — 이 뷰가 EXTEND로 확장될 수 있게 허용(예제 15에서 사용).
  • define view entity — 최신 CDS 문법. 별도 SQL 뷰 이름이 필요 없습니다.
  • key — 결과의 키. as — 출력 별칭(CamelCase 관례).

실행 방법

ADT에서 오브젝트를 열고 Data Preview(F8). 또는 Open SQL: SELECT * FROM zi_jnc_edu_01_flight INTO TABLE @DATA(lt).

02 · 계산 필드 · CASE · CAST 기본

ZI_JNC_EDU_02_CALC

목적

산술식, 타입 변환(CAST), 조건 분기(CASE), 0으로 나누기 방어를 익힙니다.

소스

define view entity ZI_JNC_EDU_02_CALC as select from sflight
{
  key carrid as AirlineID,
  key connid as ConnectionID,
  key fldate as FlightDate,
      seatsmax as SeatsMax,
      seatsocc as SeatsOccupied,
      case seatsmax
        when 0 then cast( 0 as abap.dec(15,2) )
        else cast( cast( seatsocc as abap.dec(15,2) )
                   / seatsmax * 100 as abap.dec(15,2) )
      end as OccupancyRate,
      case
        when seatsocc >= seatsmax then 'FULL'
        when seatsocc = 0         then 'EMPTY'
        else                           'AVAILABLE'
      end as SeatStatus
}

해설

  • 정수 seatsocc/seatsmax는 정수 결과가 되므로, 먼저 cast(... as dec)로 소수 연산을 만듭니다.
  • 바깥 cast(... as dec(15,2))로 소수 2자리로 반올림 → 결과가 깔끔한 96.62.
  • case seatsmax when 0 ...로 0으로 나누기를 방어합니다.

03 · JOIN (inner + left outer) 기본

ZI_JNC_EDU_03_JOIN

목적

세 테이블을 조인하고 inner와 left outer의 차이를 확인합니다.

소스

define view entity ZI_JNC_EDU_03_JOIN
  as select from sflight as f
  inner join      scarr as c on c.carrid = f.carrid
  left outer join spfli as p on  p.carrid = f.carrid
                              and p.connid = f.connid
{
  key f.carrid as AirlineID,
  key f.connid as ConnectionID,
  key f.fldate as FlightDate,
      c.carrname as AirlineName,
      p.cityfrom as CityFrom,
      p.cityto   as CityTo,
      f.price    as Price,
      f.currency as CurrencyCode
}

해설

  • inner join scarr — 항공사 마스터가 있는 행만.
  • left outer join spfli — 운항 스케줄이 없어도 항공편은 유지(CityFrom/CityTo는 NULL 가능).
  • 테이블 별칭 f/c/p로 필드 출처를 명확히.

실행 결과 (실측)

AA00172023-07-24American AirlinesNEW YORKSAN FRANCISCO423.44

04 · 집계 · GROUP BY · HAVING 기본

ZI_JNC_EDU_04_AGGR

목적

항공사별로 편수·좌석·가격을 집계하고 HAVING으로 그룹을 필터합니다.

소스

define view entity ZI_JNC_EDU_04_AGGR as select from sflight
{
  key carrid as AirlineID,
      count( * )        as FlightCount,
      sum( seatsocc )   as TotalOccupied,
      avg( cast( price as abap.dec(15,2) ) as abap.dec(15,2) ) as AvgPrice,
      max( cast( price as abap.dec(15,2) ) ) as MaxPrice,
      min( cast( price as abap.dec(15,2) ) ) as MinPrice
}
group by carrid
having count( * ) > 1

해설

  • 키가 아닌 집계 대상은 함수로 감싸고, 나머지(carrid)는 group by에.
  • 통화(CURR) 필드 price는 AVG 대상 타입에 DEC를 못 넣으므로 먼저 cast(price as dec).
  • having count(*) > 1 — 1편 이하 항공사는 제외.

05 · UNION ALL 기본

ZI_JNC_EDU_05_UNION

목적

두 SELECT 결과를 이어붙이고, 리터럴 컬럼으로 국내/국제를 구분합니다.

소스

@Metadata.ignorePropagatedAnnotations: true
define view entity ZI_JNC_EDU_05_UNION as select from spfli
{
  key carrid as AirlineID,
  key connid as ConnectionID,
      cityfrom as CityFrom,
      cityto   as CityTo,
      cast( 'DOMESTIC' as abap.char(13) ) as FlightScope
}
where countryfr = countryto
union all
select from spfli
{
  key carrid as AirlineID,
  key connid as ConnectionID,
      cityfrom as CityFrom,
      cityto   as CityTo,
      cast( 'INTERNATIONAL' as abap.char(13) ) as FlightScope
}
where countryfr <> countryto

해설

  • UNION의 두 브랜치는 컬럼 개수·타입·순서가 일치해야 합니다.
  • 구분 리터럴은 길이가 같아야 하므로 char(13)으로 캐스트(가장 긴 'INTERNATIONAL'=13자).
  • View Entity의 UNION에는 @Metadata.ignorePropagatedAnnotations: true가 필수.

06 · 파라미터 뷰 · 세션 변수 기본

ZI_JNC_EDU_06_PARAM

목적

입력 파라미터로 뷰를 필터하고, 세션 변수로 현재 날짜를 가져옵니다.

소스

define view entity ZI_JNC_EDU_06_PARAM
  with parameters
    P_MinSeats : abap.int4
  as select from sflight
{
  key carrid as AirlineID,
  key connid as ConnectionID,
  key fldate as FlightDate,
      seatsmax as SeatsMax,
      seatsocc as SeatsOccupied,
      cast( $session.system_date as abap.dats ) as QueryDate
}
where seatsmax >= $parameters.P_MinSeats

해설

  • with parameters P_MinSeats : abap.int4 — 입력 파라미터 선언.
  • 본문에서 $parameters.P_MinSeats로 참조(옛 :P_MinSeats는 View Entity에서 오류).
  • $session.system_date — 현재 시스템 날짜.

07 · Association 기본 기본

ZI_JNC_EDU_07_ASSOC

목적

association을 정의하고 경로식으로 관련 필드를 가져오며, association 자체를 노출합니다.

소스

define view entity ZI_JNC_EDU_07_ASSOC
  as select from sflight
  association [1..1] to scarr as _Carrier
    on $projection.AirlineID = _Carrier.carrid
{
  key carrid as AirlineID,
  key connid as ConnectionID,
  key fldate as FlightDate,
      price    as Price,
      currency as CurrencyCode,
      _Carrier.carrname as CarrierName,
      _Carrier
}

해설

  • association [1..1] to scarr as _Carrier — 항공편 1건당 항공사 1건.
  • on $projection.AirlineID = _Carrier.carrid — 자기 뷰 필드는 $projection.별칭.
  • _Carrier.carrname 경로식 사용 + _Carrier 노출(상위 뷰가 재사용 가능).
  • CURR 필드 price를 노출하므로 currency도 함께 노출(안 하면 오류).

실행 결과 (실측)

AirlineIDConnectionIDFlightDatePriceCurrencyCodeCarrierName
AA00172023-06-22423.44USDAmerican Airlines
직접 바꿔보기 — _Carrier.currcode as CarrierCurrency를 추가해 항공사 통화도 가져와 보세요.

08 · Association 재사용 기본

ZI_JNC_EDU_08_ASSOC_EXPOSE

목적

하위 뷰(07)가 노출한 association을 상위 뷰에서 다시 정의하지 않고 재사용합니다.

소스

define view entity ZI_JNC_EDU_08_ASSOC_EXPOSE
  as select from ZI_JNC_EDU_07_ASSOC as base
{
  key base.AirlineID,
  key base.ConnectionID,
  key base.FlightDate,
      base.Price,
      base.CurrencyCode,
      base._Carrier.carrname as CarrierName,
      base._Carrier.currcode as CarrierCurrency
}

해설

  • 소스가 테이블이 아니라 다른 CDS 뷰(07).
  • base._Carrier.currcode — 07이 노출한 association을 그대로 사용. 관계를 재정의할 필요가 없습니다.
  • 이것이 VDM에서 인터페이스 뷰를 재사용하는 기본 패턴입니다.

09 · 필터드 Association · 스칼라 경로식 심화

ZI_JNC_EDU_09_ASSOC_FILTER

목적

to-many association에 필터+카디널리티를 지정해 단일 값을 꺼냅니다.

소스

define view entity ZI_JNC_EDU_09_ASSOC_FILTER
  as select from sflight as f
  association [0..*] to sbook as _Bookings
    on  $projection.AirlineID    = _Bookings.carrid
    and $projection.ConnectionID = _Bookings.connid
    and $projection.FlightDate   = _Bookings.fldate
{
  key f.carrid as AirlineID,
  key f.connid as ConnectionID,
  key f.fldate as FlightDate,
      f.seatsocc as SeatsOccupied,
      _Bookings[1: bookid = '00000001'].custtype as FirstBookingCustType,
      _Bookings
}

해설

  • [0..*] to sbook — 항공편당 예약 여러 건(to-many).
  • _Bookings[1: bookid = '00000001'] — 필터로 1건만 선택 → 스칼라 필드 .custtype 추출 가능.
  • to-many를 그대로 스칼라로 쓸 수 없고, 반드시 카디널리티/필터로 1건을 지정해야 합니다.

직접 바꿔보기 — 필터 조건을 [1: bookid = '00000002']로 바꿔 다른 예약의 고객 유형을 가져와 보세요.

10 · 통화 시맨틱 심화

ZI_JNC_EDU_10_AMOUNT

목적

금액 필드가 어떤 통화 필드와 짝인지 @Semantics.amount.currencyCode로 명시합니다.

소스

define view entity ZI_JNC_EDU_10_AMOUNT as select from sbook
{
  key carrid as AirlineID,
  key connid as ConnectionID,
  key fldate as FlightDate,
  key bookid as BookingID,
      @Semantics.amount.currencyCode: 'LocalCurrency'
      loccuram  as LocalAmount,
      loccurkey as LocalCurrency,
      @Semantics.amount.currencyCode: 'ForeignCurrency'
      forcuram  as ForeignAmount,
      forcurkey as ForeignCurrency

해설

  • 어노테이션 값 'LocalCurrency'는 같은 뷰의 통화 필드 별칭을 가리킵니다.
  • 이렇게 짝을 지어주면 프레임워크가 금액을 통화에 맞게 처리·표시합니다.

11–13 · VDM 3계층 (Basic → Composite → Consumption) 심화

ZP_JNC_EDU_11_FLIGHT · ZI_JNC_EDU_12_FLIGHTCARR · ZC_JNC_EDU_13_FLIGHTOVW

목적

하나의 시나리오를 세 계층으로 나누어, 아래층을 재사용하며 위로 조합하는 VDM 패턴을 익힙니다.

계층 흐름 — sflight → 11 Basic(1:1 투영+association) → 12 Composite(association 결합+파생 필드) → 13 Consumption(필드 선별+라벨).

11 · Basic (ZP_JNC_EDU_11_FLIGHT)

@VDM.viewType: #BASIC
define view entity ZP_JNC_EDU_11_FLIGHT
  as select from sflight
  association [1..1] to scarr as _Carrier
    on $projection.AirlineID = _Carrier.carrid
{
  key carrid as AirlineID,
  key connid as ConnectionID,
  key fldate as FlightDate,
      price as Price, currency as CurrencyCode,
      seatsmax as SeatsMax, seatsocc as SeatsOccupied,
      _Carrier
}

기반 뷰는 ZP_(private)로 두어 내부 전용임을 표시. 로직 없이 1:1 투영 + association만.

12 · Composite (ZI_JNC_EDU_12_FLIGHTCARR)

@VDM.viewType: #COMPOSITE
define view entity ZI_JNC_EDU_12_FLIGHTCARR
  as select from ZP_JNC_EDU_11_FLIGHT as base
{
  key base.AirlineID, key base.ConnectionID, key base.FlightDate,
      base._Carrier.carrname as AirlineName,
      base.Price, base.CurrencyCode,
      case when base.SeatsMax = 0 then cast( 0 as abap.dec(15,2) )
           else cast( cast( base.SeatsOccupied as abap.dec(15,2) )
                      / base.SeatsMax * 100 as abap.dec(15,2) ) end as OccupancyRate
}

기반 뷰(11)를 select from하고 그 association·필드를 재사용하며, 파생 필드(OccupancyRate)를 계산.

13 · Consumption (ZC_JNC_EDU_13_FLIGHTOVW)

@VDM.viewType: #CONSUMPTION
define view entity ZC_JNC_EDU_13_FLIGHTOVW
  as select from ZI_JNC_EDU_12_FLIGHTCARR as comp
{
  key comp.AirlineID    as Airline,
  key comp.ConnectionID as ConnectionID,   // 'Connection'은 예약어!
  key comp.FlightDate   as FlightDate,
      comp.AirlineName, comp.Price, comp.CurrencyCode, comp.OccupancyRate
}

소비 뷰는 ZC_. 앱/OData가 쓸 필드만 선별·정리. 주의: Connection은 예약어라 별칭 불가 → ConnectionID.

14 · Analytics CUBE 심화

ZI_JNC_EDU_14_ANALYTICS

목적

분석용 CUBE 뷰로 측정값과 기본 집계를 지정합니다.

소스

@Analytics.dataCategory: #CUBE
define view entity ZI_JNC_EDU_14_ANALYTICS as select from sflight
  association [1..1] to scarr as _Carrier
    on $projection.AirlineID = _Carrier.carrid
{
  key carrid as AirlineID, key connid as ConnectionID, key fldate as FlightDate,
      @DefaultAggregation: #SUM
      seatsocc as SeatsOccupied,
      @Semantics.amount.currencyCode: 'CurrencyCode'
      @DefaultAggregation: #SUM
      paymentsum as PaymentSum,
      currency as CurrencyCode,
      _Carrier
}

해설

  • @Analytics.dataCategory: #CUBE — 이 뷰를 측정값 큐브로 표시.
  • @DefaultAggregation: #SUM — 측정값의 기본 집계 방식.
  • 키 필드는 차원(dimension), 집계 대상은 측정값(measure)으로 소비됩니다.

17 · Table Function + AMDP 심화

ZI_JNC_EDU_17_TABFUNC + ZCL_JNC_EDU_17_TABFUNC

목적

순수 CDS로 어려운 로직을 Table Function으로 선언하고 AMDP(SQLScript)로 구현합니다.

소스 — Table Function

define table function ZI_JNC_EDU_17_TABFUNC
  returns
  {
    client       : abap.clnt;
    airlineid    : s_carr_id;
    connectionid : s_conn_id;
    flightcount  : abap.int4;
    totalpayment : abap.dec(17,2);
  }
  implemented by method ZCL_JNC_EDU_17_TABFUNC=>get_flight_stats;

소스 — AMDP 클래스

CLASS zcl_jnc_edu_17_tabfunc DEFINITION PUBLIC FINAL CREATE PUBLIC.
  PUBLIC SECTION.
    INTERFACES if_amdp_marker_hdb.
    CLASS-METHODS get_flight_stats
      FOR TABLE FUNCTION ZI_JNC_EDU_17_TABFUNC.
ENDCLASS.
CLASS zcl_jnc_edu_17_tabfunc IMPLEMENTATION.
  METHOD get_flight_stats
      BY DATABASE FUNCTION FOR HDB LANGUAGE SQLSCRIPT
      OPTIONS READ-ONLY USING sflight.
    RETURN
      SELECT mandt AS client, carrid AS airlineid, connid AS connectionid,
             COUNT(*) AS flightcount, SUM( paymentsum ) AS totalpayment
        FROM sflight GROUP BY mandt, carrid, connid;
  ENDMETHOD.
ENDCLASS.

해설

  • client-specific이면 RETURNS 첫 필드에 client : abap.clnt가 필요하고, AMDP는 SELECT mandt AS client ....
  • AMDP 메서드는 BY DATABASE FUNCTION FOR HDB LANGUAGE SQLSCRIPT + USING 절에 참조 테이블.
  • 활성화 순서: AMDP 클래스와 Table Function은 함께 활성화됩니다.
  • 호출은 괄호 필수: SELECT * FROM zi_jnc_edu_17_tabfunc( ).

18 · Hierarchy 심화

ZI_JNC_EDU_18_HIER + ZI_JNC_EDU_18_HIERSRC

목적

부모-자식 자기참조 association으로 2단계 계층(항공사 → 항공편)을 만듭니다.

소스 — 계층 소스 뷰

@Metadata.ignorePropagatedAnnotations: true
define view entity ZI_JNC_EDU_18_HIERSRC
  as select from scarr
  association [0..1] to ZI_JNC_EDU_18_HIERSRC as _Parent
    on $projection.ParentId = _Parent.NodeId
{
  key cast( carrid as abap.char(20) )    as NodeId,
      cast( '' as abap.char(20) )        as ParentId,
      cast( carrname as abap.char(30) )  as NodeName,
      cast( 'CARRIER' as abap.char(10) ) as NodeType, _Parent
}
union all
select from spfli
  association [0..1] to ZI_JNC_EDU_18_HIERSRC as _Parent
    on $projection.ParentId = _Parent.NodeId
{
  key cast( concat( carrid, cast( connid as abap.char(4) ) ) as abap.char(20) ) as NodeId,
      cast( carrid as abap.char(20) )   as ParentId,
      cast( cityto as abap.char(30) )   as NodeName,
      cast( 'FLIGHT' as abap.char(10) ) as NodeType, _Parent
}

소스 — Hierarchy

define hierarchy ZI_JNC_EDU_18_HIER
  as parent child hierarchy(
    source ZI_JNC_EDU_18_HIERSRC
    child to parent association _Parent
    start where ParentId = ' '
    siblings order by NodeId
  )
{ key NodeId, ParentId, NodeName, NodeType }

해설

  • 소스는 scarr(루트=항공사) ∪ spfli(자식=항공편). 자식의 ParentId=carrid로 부모를 지정.
  • start where ParentId = ' ' — 루트 노드(부모 없음)에서 시작.
  • 활성화하면 계층 순위/레벨 같은 pseudo-column을 계층 소비 시 사용할 수 있습니다.
