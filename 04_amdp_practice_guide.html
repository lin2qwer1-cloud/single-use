1장 · A1 BASICS — AMDP 기본 구조 기본

ZCL_JNC_EDU_AMDP_01_BASICS

클래스 헤더에 공용 타입(ty_amount, tt_flight 등)을 선언하고, 세 메서드가 이를 씁니다.

A1-1 · 최소 AMDP 프로시저

목적

가장 단순한 프로시저 — 값 전달 파라미터, USING, READ-ONLY, SQLScript에서 :iv_carrid 참조.

소스

CLASS-METHODS get_flights_by_carrier
  IMPORTING VALUE(iv_carrid)  TYPE s_carr_id
  EXPORTING VALUE(et_flights) TYPE tt_flight.
...
METHOD get_flights_by_carrier
    BY DATABASE PROCEDURE FOR HDB LANGUAGE SQLSCRIPT
    OPTIONS READ-ONLY USING sflight.
  et_flights = SELECT carrid, connid, fldate, price, currency, seatsocc
                 FROM sflight
                WHERE mandt  = SESSION_CONTEXT('CLIENT')
                  AND carrid = :iv_carrid
                ORDER BY connid, fldate;
ENDMETHOD.

해설

  • 결과 테이블 et_flights에 SELECT 결과를 통째로 대입.
  • SESSION_CONTEXT('CLIENT')로 현재 클라이언트 필터(AMDP는 자동 필터 없음).
  • ABAP 파라미터 iv_carrid → SQLScript에서 :iv_carrid.

실행 결과 (실측)

get_flights_by_carrier('AA') → 454행, 첫 행 [AA/0017 2020-01-01 price=1000.00 occ=0] (2020년은 0원 템플릿 행).

직접 바꿔보기 — AND paymentsum > 0을 추가해 실제 운항 행만 반환해 보세요.

A1-2 · 조인 + 집계, 필터 조기 적용

목적

조인·집계 결과를 EXPORTING 테이블로. 파라미터 필터를 HAVING에.

소스

METHOD get_flight_revenue
    BY DATABASE PROCEDURE FOR HDB LANGUAGE SQLSCRIPT
    OPTIONS READ-ONLY USING sflight scarr.
  et_revenue = SELECT f.carrid, f.connid, c.carrname,
                      SUM( f.paymentsum ) AS totalpay,
                      COUNT(*)            AS flightcnt
                 FROM sflight AS f
                 INNER JOIN scarr AS c
                   ON c.mandt = f.mandt AND c.carrid = f.carrid
                WHERE f.mandt = SESSION_CONTEXT('CLIENT')
                GROUP BY f.carrid, f.connid, c.carrname
               HAVING SUM( f.paymentsum ) >= :iv_min_paysum
                ORDER BY totalpay DESC;
ENDMETHOD.

해설

  • 조인 조건에 mandt도 포함(클라이언트 정합성).
  • 집계 후 조건은 HAVING. 파라미터 :iv_min_paysum 사용.

실행 결과 (실측)

get_flight_revenue('1000000.00') → 23행. 상위:

carrid/connidcarrnametotalpaycnt
SQ/0015Singapore Airlines11752049.4113
SQ/0002Singapore Airlines11353954.0413
JL/0407Japan Airlines5080557.9713
직접 바꿔보기 — iv_min_paysum을 '5000000.00'으로 올려 행 수 변화를 관찰하세요.

A1-3 · AMDP가 AMDP를 호출

목적

같은 클래스의 다른 AMDP를 USING + CALL로 재사용.

소스

METHOD get_carrier_stats
    BY DATABASE PROCEDURE FOR HDB LANGUAGE SQLSCRIPT
    OPTIONS READ-ONLY
    USING zcl_jnc_edu_amdp_01_basics=>get_flights_by_carrier.
  CALL "ZCL_JNC_EDU_AMDP_01_BASICS=>GET_FLIGHTS_BY_CARRIER"(
    iv_carrid  => :iv_carrid,
    et_flights => lt_flights );
  et_stats = SELECT carrid,
                    COUNT(*)                            AS flights,
                    AVG( TO_DECIMAL( seatsocc, 9, 2 ) ) AS avgocc
               FROM :lt_flights
              GROUP BY carrid;
ENDMETHOD.

해설

  • USING 클래스=>메서드로 호출 대상 AMDP를 명시.
  • SQLScript에서 CALL "CLASS=>METHOD"(...), 출력은 로컬 테이블 변수 lt_flights로 받아 이어서 집계.

실행 결과 (실측)

get_carrier_stats('LH') → [LH flights=105 avgocc=154.57]

직접 바꿔보기 — get_flights_by_carrier에 paymentsum>0 필터를 넣으면 A1-3의 avgocc가 어떻게 바뀌나요?

ZCL_JNC_EDU_AMDP_02_SQLSCRIPT

A2-1 · 선언형 체이닝 (필터→조인→집계) 기본

목적

중간 결과를 테이블 변수로 이어붙이며 3단계로 표현.

소스

METHOD chain_filter_aggregate ... USING sflight scarr.
  lt_flights = SELECT mandt, carrid, connid, paymentsum
                 FROM sflight
                WHERE mandt = SESSION_CONTEXT('CLIENT')
                  AND seatsocc >= :iv_min_occ;        -- 1단계: 필터 조기
  lt_joined  = SELECT f.carrid, c.carrname, f.paymentsum
                 FROM :lt_flights AS f                -- :lt_flights 재사용
                 INNER JOIN scarr AS c
                   ON c.mandt = f.mandt AND c.carrid = f.carrid;
  et_chain   = SELECT carrid, carrname, SUM( paymentsum ) AS totalpay
                 FROM :lt_joined                      -- 3단계: 집계는 마지막
                GROUP BY carrid, carrname
                ORDER BY totalpay DESC;
ENDMETHOD.

해설

  • 필터를 첫 SELECT에 적용해 이후 단계의 데이터량을 줄임(품질 원칙 ③).
  • 각 단계는 앞 단계 결과 :lt_x를 입력으로. 엔진이 전체를 한 번에 최적화.

실행 결과 (실측)

chain_filter_aggregate(iv_min_occ=300) → 8행. 상위: SQ Singapore Airlines tot=23401273.48, AZ Alitalia tot=12438092.58.

직접 바꿔보기 — 2단계와 3단계를 하나의 SELECT로 합쳐 보고, 가독성을 비교하세요.

A2-3 · 윈도우 함수 RANK() — 그룹별 Top-N 심화

목적

항공사별 매출 순위를 매기고 Top-3만 추출.

소스

lt_ranked = SELECT carrid, connid, fldate, paymentsum AS paysum,
                   RANK() OVER( PARTITION BY carrid
                                ORDER BY paymentsum DESC ) AS rnk
              FROM sflight
             WHERE mandt = SESSION_CONTEXT('CLIENT');
et_rank   = SELECT carrid, connid, fldate, paysum, rnk
              FROM :lt_ranked
             WHERE rnk <= :iv_top_n            -- Top-N은 바깥에서 필터
             ORDER BY carrid, rnk;

해설

  • PARTITION BY carrid로 항공사별 그룹, 그룹 안에서 paymentsum DESC 순위.
  • 순위 계산 후 바깥 SELECT에서 rnk <= N으로 Top-N. GROUP BY와 달리 행이 유지됨.

실행 결과 (실측)

rank_top_flights(iv_top_n=3) → 24행(8항공사×3).

carridrnkconnid/fldatepaysum

AA20017 / 2023-12-31193474.13
AA30017 / 2023-11-29193359.85
AZ10789 / 2023-08-25553738.30
직접 바꿔보기 — RANK()를 ROW_NUMBER()나 DENSE_RANK()로 바꿔 동점 처리 차이를 관찰하세요.

A2-4 · 누적합 SUM() OVER + LAG() 심화

소스

et_running = SELECT carrid, fldate, paymentsum AS paysum,
                    SUM( paymentsum ) OVER( PARTITION BY carrid
                                            ORDER BY fldate ) AS cumsum,
                    LAG( paymentsum ) OVER( PARTITION BY carrid
                                            ORDER BY fldate ) AS prevpay
               FROM sflight
              WHERE mandt  = SESSION_CONTEXT('CLIENT')
                AND carrid = :iv_carrid
                AND paymentsum > 0              -- 0원 템플릿 행 제외
              ORDER BY fldate
              LIMIT 15;

해설

  • SUM() OVER(ORDER BY fldate) — 날짜 순 누적합.
  • LAG() — 같은 그룹의 이전 행 값(첫 행은 NULL→0).
  • paymentsum > 0으로 2020년 0원 템플릿 행을 걸러 누적이 실제로 증가하게 함.

실행 결과 (실측, 'AA')

fldatepaysumcumsumprevpay(LAG)
2023-06-22191452.52191452.520.00
2023-06-24167962.31359414.83191452.52
2023-07-24190737.65550152.48167962.31
2023-07-26167754.97717907.45190737.65

cumsum이 계속 누적되고, prevpay가 직전 행의 paysum과 일치하는 것을 확인하세요.

직접 바꿔보기 — LEAD()로 다음 행 값을 가져오는 컬럼을 추가해 보세요.

3장 · A3 IMPERATIVE — 명령형 & 예외 심화

ZCL_JNC_EDU_AMDP_03_IMPERATIVE

A3-1 vs A3-2 · 커서 루프(나쁜 예) vs 선언형(좋은 예)

목적

같은 결과를 두 방식으로 구현하고, 러너로 나란히 실행해 결과가 동일함을 확인.

✗ A3-1 CURSOR_LOOP_BAD
DECLARE lv_carrid NVARCHAR(3) := '';
DECLARE lv_total  DECIMAL(11,2) := 0;
DECLARE CURSOR lc_flights FOR
  SELECT carrid, paymentsum FROM sflight
   WHERE mandt = SESSION_CONTEXT('CLIENT')
   ORDER BY carrid;
et_sum = SELECT carrid, TO_DECIMAL(0,11,2) AS totalpay
           FROM sflight WHERE 1 = 0;
FOR ls_row AS lc_flights DO
  IF :lv_carrid <> '' AND :lv_carrid <> ls_row.carrid THEN
    et_sum = SELECT * FROM :et_sum UNION ALL
             SELECT :lv_carrid, :lv_total FROM DUMMY;
    lv_total := 0;
  END IF;
  lv_carrid := ls_row.carrid;
  lv_total  := :lv_total + ls_row.paymentsum;
END FOR;
-- 마지막 그룹 flush ...
✓ A3-2 DECLARATIVE_GOOD
et_sum = SELECT carrid,
                SUM( paymentsum ) AS totalpay
           FROM sflight
          WHERE mandt = SESSION_CONTEXT('CLIENT')
          GROUP BY carrid
          ORDER BY carrid;

커서 루프 20여 줄이 GROUP BY 한 문장으로. 엔진이 병렬 집계.

A3-3 · 예외 처리 (EXIT HANDLER)

소스

METHOD error_handling ... USING sflight.
  DECLARE EXIT HANDLER FOR SQLEXCEPTION
    BEGIN
      ev_result := -1;
      ev_errmsg := SUBSTRING( ::SQL_ERROR_MESSAGE, 1, 100 );
    END;
  ev_errmsg := '';
  IF :iv_force_error = 'X' THEN
    SELECT 1 / 0 INTO ev_result FROM DUMMY;      -- 의도적 오류
  ELSE
    SELECT COUNT(*) INTO ev_result FROM sflight
     WHERE mandt = SESSION_CONTEXT('CLIENT');
  END IF;
ENDMETHOD.

해설

  • DECLARE EXIT HANDLER FOR SQLEXCEPTION — SQL 오류 시 핸들러로 진입해 ev_result=-1 설정.
  • ABAP 측에서는 (핸들러가 없다면) cx_amdp_execution_failed로 올라옵니다.
실전 함정 — ::SQL_ERROR_MESSAGE는 156자였습니다. c100 필드에 그대로 넣으면 대입 자체가 2차 오류(exceeds maximum length by 56)를 냅니다. 반드시 SUBSTRING(...,1,100)으로 절단하세요.

실행 결과 (실측)

호출ev_resultev_errmsg
error_handling(false)870(없음)
error_handling(true)-1division by zero undefined: ...
직접 바꿔보기 — 1/0을 존재하지 않는 컬럼 참조 등 다른 오류로 바꿔 메시지가 어떻게 달라지는지 보세요.

4장 · A4 INTEGRATION — CDS 연계 & 클라이언트 심화

ZCL_JNC_EDU_AMDP_04_INTEG + ZI_JNC_EDU_AMDP_TF_TOPN
네이밍 참고 — ABAP 클래스명은 최대 30자입니다. ZCL_JNC_EDU_AMDP_04_INTEGRATION은 31자라 한도를 넘어, 이 예제에서는 ZCL_JNC_EDU_AMDP_04_INTEG로 줄였습니다. (다른 클래스 01~03은 30자 이내라 그대로 사용.)

A4-1 · 파라미터 있는 CDS Table Function

목적

CDS 파라미터(p_top_n)를 AMDP 함수로 전달해 항공사별 Top-N을 반환.

소스 — CDS Table Function

define table function ZI_JNC_EDU_AMDP_TF_TOPN
  with parameters p_top_n : abap.int4
  returns { client : abap.clnt; airlineid : s_carr_id; connectionid : s_conn_id;
            flightdate : s_date; paymentsum : abap.dec(17,2); rnk : abap.int4; }
  implemented by method ZCL_JNC_EDU_AMDP_04_INTEG=>get_top_flights;

소스 — AMDP 함수

METHOD get_top_flights
    BY DATABASE FUNCTION FOR HDB LANGUAGE SQLSCRIPT
    OPTIONS READ-ONLY USING sflight.
  RETURN
    SELECT client, airlineid, connectionid, flightdate, paymentsum, rnk
      FROM ( SELECT mandt AS client, carrid AS airlineid, connid AS connectionid,
                    fldate AS flightdate, paymentsum,
                    RANK() OVER( PARTITION BY mandt, carrid
                                 ORDER BY paymentsum DESC ) AS rnk
               FROM sflight )
     WHERE rnk <= :p_top_n                 -- CDS 파라미터 → :p_top_n
     ORDER BY airlineid, rnk;
ENDMETHOD.

해설

  • BY DATABASE FUNCTION + FOR TABLE FUNCTION — 함수형 AMDP.
  • CDS의 p_top_n이 SQLScript의 :p_top_n으로 그대로 전달됨.
  • client-specific이므로 RETURNS 첫 필드는 client : abap.clnt(CDS 예제 17과 동일 규칙).

실행 방법 / 결과 (실측)

A4-2 · 클라이언트 처리

소스

CLASS-METHODS client_handling
  AMDP OPTIONS CDS SESSION CLIENT CURRENT      -- 현재 클라이언트 주입
  EXPORTING VALUE(et_cnt) TYPE tt_cnt.
...
et_cnt = SELECT 'CDS_CLIENT filter' AS clientmode, COUNT(*) AS cnt
           FROM sflight
          WHERE mandt = SESSION_CONTEXT('CDS_CLIENT')
         UNION ALL
         SELECT 'no client filter' AS clientmode, COUNT(*) AS cnt
           FROM sflight;

해설

  • AMDP OPTIONS CDS SESSION CLIENT CURRENT — 현재 클라이언트를 SESSION_CONTEXT('CDS_CLIENT')로 주입.
  • 필터를 빠뜨리면 모든 클라이언트의 데이터가 집계됩니다.

실행 결과 (실측)

clientmodecnt
CDS_CLIENT filter870
no client filter886

현재 클라이언트(100)는 870행, 무필터는 886행 — 타 클라이언트 16행이 섞입니다. 클라이언트 필터는 선택이 아니라 필수입니다.

직접 바꿔보기 — SESSION_CONTEXT('CDS_CLIENT')를 SESSION_CONTEXT('CLIENT')로 바꿔도 같은 값인지 확인하세요.
