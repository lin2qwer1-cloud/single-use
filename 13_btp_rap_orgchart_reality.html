1부. 이 구성은 얼마나 "정석"인가 기초

결론: 골격은 SAP 공식 튜토리얼이 가르치는 표준 패턴 그대로입니다.

  • 계층 구조 — 루트(ZR_) 하나에 용도별 소비뷰(ZC_)를 여럿 얹는 "R→C 다중 프로젝션"은 RAP 마법사가 자동 생성하는 바로 그 구조입니다. 조회 서비스와 관리 서비스를 분리한 것, 값도움말을 전용 뷰(ZI_…_VH)로 뺀 것도 관례 그대로.
  • managed + draft — S/4HANA Cloud의 표준 앱들이 실제로 이 조합으로 만들어져 있습니다. 비드래프트를 고집하다 Create 버튼 없는 화면을 만난 것 자체가 "왜 표준이 draft인가"의 답이었습니다.
  • 검증의 위치 — 규칙 선언은 BDEF, 구현은 핸들러, 메시지는 T100 메시지 클래스. UI에서도 막고(값도움말) 저장에서도 막는(validation) 이중 방어까지 포함해 교과서적 배치입니다.
  • 엑셀 업로드 패턴 — largeObject 파일 BO + XCO XLSX + cross-BO EML은 SAP 공식 블로그가 BTP ABAP 환경용으로 안내하는 그 패턴입니다. 특히 EML로 넣어 검증이 우회되지 않게 한 것은 실무에서도 그대로 칭찬받을 선택입니다.
  • VDM/네이밍 — 단일 테이블·단일 소비 시나리오라 I_ 인터페이스 계층은 일부러 넣지 않았습니다 (과설계 회피). I_가 필요해지는 건 "여러 소비자가 같은 조인/계산 로직을 재사용할 때"부터입니다.

2부. 데모라서 눈감은 곳 + 플랫폼 한계 기초

2.1 실무 프로젝트였다면 리뷰에서 지적됐을 것들

항목데모의 현재실무의 정석
권한all-allow 스텁 (전부 허용)권한 오브젝트 정의 + AUTHORITY-CHECK 실구현, IAM 앱/비즈니스 카탈로그로 역할 배포
테스트없음validation·액션 로직에 ABAP Unit (EML 테스트 더블/BO 테스트 프레임워크)

2.2 플랫폼 자체의 한계 — 우리가 어쩔 수 없던 것

  • FE V4에 표준 엑셀 Import가 없다. Export는 표준, Import는 지금도 커스텀의 영역. 그래서 업로드 BO 패턴이 "우회"가 아니라 "현재의 표준 답"입니다.
  • 비드래프트 V4 UI에는 Create/Edit UX가 없다 — 백엔드 능력과 무관한 프런트 정책입니다.
  • ADT Preview는 런치패드가 아니다 — 앱 간 이동(IBN), 화면 통합, 커스텀 툴바 버튼은 Preview 밖(정식 배포)의 세계입니다.
  • Trial 특성 — 공유 인스턴스(오브젝트 공개), UCON의 외부 API 차단, 90일 소멸. 학습엔 사소하지만 "여기 있는 건 언제든 사라진다"는 전제는 항상 유효합니다.

3부. 실무의 기대, 실무의 구성 기초

"SAP GUI 시절엔 한 화면에서 다 됐는데요?" — 반드시 듣게 될 그 말에 대하여.

GUI 세대 사용자의 기대는 명확합니다: 조회하다가 그 화면에서 바로 고치고, 같은 화면의 버튼으로 엑셀을 올리는 것. ALV Grid + 업로드 버튼으로 20년을 살아온 사용자에게 "조회 앱 따로, 관리 앱 따로, 업로드 앱 따로"는 퇴보로 보입니다. 이 갈등은 S/4 전환 프로젝트의 단골 쟁점이고, 업계의 현실적인 대응은 세 갈래입니다.

전략내용비용/비고
① 커스텀 Fiori 앱 BAS(Business Application Studio)에서 Fiori Elements 앱을 정식 생성, 컨트롤러 익스텐션으로 List Report 툴바에 업로드 버튼 삽입 — 오픈소스 UI5 Spreadsheet Importer가 사실상의 커뮤니티 표준. 사용자 기대에 가장 근접. 프런트 개발·배포 파이프라인 필요. RAP 백엔드는 지금 것 그대로 재사용
② 표준 UX 설득 "업로드는 업로드 앱, 관리는 관리 앱"이라는 Fiori 철학대로 가고 사용자를 교육. 클린코어 관점에선 가장 순수. 개발비 0, 변화관리 비용은 최대. 사용자 만족도 리스크
③ IBN 절충 앱은 분리하되 Intent-Based Navigation 버튼으로 한 클릭 이동(조회→관리→업로드). 런치패드의 Semantic Object 타겟 매핑으로 구현. 어노테이션+런치패드 설정 수준. 단 Preview에선 동작 안 함(런치패드 필요)
실무 구성 예시를 하나로 그리면 — 백엔드는 지금 만든 RAP 스택 그대로 (루트+검증+업로드 BO), 그 위에 BAS로 만든 Fiori 앱 하나가 List Report(트리 or 목록)를 메인으로 삼고, 툴바에 Spreadsheet Importer 업로드 버튼 + IBN 상세 이동을 얹습니다. 권한은 IAM 앱·비즈니스 카탈로그로 역할화하고, 전송은 gCTS 트랙을 탑니다. 즉 이번에 만든 것은 버려지는 연습작이 아니라, 실무 구성의 백엔드 절반이 이미 완성된 상태입니다.

4부. 이후 작업 — 구체적으로 심화

  1. BAS Fiori 앱 프로젝트 (다음 라운드의 본론). Trial의 Business Application Studio 구독을 켜고 지금 서비스(ZUI_JMG_ORG_MAINT_O4)를 소비하는 List Report 앱 생성 → ① 트리 화면의 manifest에 type: "TreeTable" + hierarchyQualifier를 넣어 정식 트리 테이블 구성 ② Spreadsheet Importer로 툴바 업로드 버튼 ③ Semantic Object 등록 후 조회→관리→업로드 IBN 버튼 연결. 이번에 Preview 한계로 미룬 UX 세 가지가 전부 여기서 풀립니다.
  2. 권한 실구현. all-allow 스텁을 걷어내고 권한 오브젝트 + AUTHORITY-CHECK, IAM 앱 구성까지. RAP 권한 3종(global/instance/precheck)을 한 번씩 실습하기 좋은 소재입니다.
  3. 업로드 고도화. 행 단위 오류 리포트(결과 로그 엔티티), 멱등 업서트 모드, 템플릿 파일 다운로드 액션.
  4. 순환참조 완전 차단. SELECT … FROM HIERARCHY로 조상 목록을 얻어 "새 부모가 내 자손이면 거부"를 validation에 추가 — CDS Hierarchy를 검증 로직에 재사용하는 좋은 연습이 됩니다.
  5. ABAP Unit. validation 두 개부터 테스트 클래스를 붙여 RAP 테스트 프레임워크 입문.
  6. 90일 대비 재생성 리허설. 인스턴스가 사라졌다 치고, 아래 부록의 소스만으로 새 trial에 전체 스택을 다시 세워보기 — 백업의 완전성을 증명하는 가장 확실한 방법입니다.

CLASS lhc_ZR_JMG_ORG_NODE DEFINITION INHERITING FROM cl_abap_behavior_handler.
  PRIVATE SECTION.
    METHODS checkSelfParent FOR VALIDATE ON SAVE
      IMPORTING keys FOR ZR_JMG_ORG_NODE~checkSelfParent.

    METHODS checkParentExists FOR VALIDATE ON SAVE
      IMPORTING keys FOR ZR_JMG_ORG_NODE~checkParentExists.

    METHODS get_global_authorizations FOR GLOBAL AUTHORIZATION
      IMPORTING REQUEST requested_authorizations FOR ZR_JMG_ORG_NODE RESULT result.

    METHODS get_instance_authorizations FOR INSTANCE AUTHORIZATION
      IMPORTING keys REQUEST requested_authorizations FOR ZR_JMG_ORG_NODE RESULT result.
ENDCLASS.

CLASS lhc_ZR_JMG_ORG_NODE IMPLEMENTATION.
  METHOD checkSelfParent.
    READ ENTITIES OF ZR_JMG_ORG_NODE IN LOCAL MODE
      ENTITY ZR_JMG_ORG_NODE
        FIELDS ( NodeID ParentID )
        WITH CORRESPONDING #( keys )
      RESULT DATA(lt_node).

    LOOP AT lt_node INTO DATA(ls_node).
      IF ls_node-NodeID = ls_node-ParentID.
        APPEND VALUE #( %tky = ls_node-%tky ) TO failed-ZR_JMG_ORG_NODE.
        APPEND VALUE #( %tky = ls_node-%tky
                         %msg = NEW_MESSAGE( id       = 'ZJMG_ORG_MSG'
                                              number   = '001'
                                              v1       = ls_node-NodeID
                                              severity = if_abap_behv_message=>severity-error ) )
               TO reported-ZR_JMG_ORG_NODE.
      ENDIF.
    ENDLOOP.
  ENDMETHOD.

  METHOD checkParentExists.
    READ ENTITIES OF ZR_JMG_ORG_NODE IN LOCAL MODE
      ENTITY ZR_JMG_ORG_NODE
        FIELDS ( NodeID ParentID )
        WITH CORRESPONDING #( keys )
      RESULT DATA(lt_parent_check).

    LOOP AT lt_parent_check INTO DATA(ls_check) WHERE ParentID IS NOT INITIAL.
      SELECT SINGLE @abap_true
        FROM zjmg_org_node
        WHERE node_id = @ls_check-ParentID
        INTO @DATA(lv_exists).

      IF lv_exists <> abap_true.
        APPEND VALUE #( %tky = ls_check-%tky ) TO failed-ZR_JMG_ORG_NODE.
        APPEND VALUE #( %tky = ls_check-%tky
                         %msg = NEW_MESSAGE( id       = 'ZJMG_ORG_MSG'
                                              number   = '002'
                                              v1       = ls_check-ParentID
                                              severity = if_abap_behv_message=>severity-error ) )
               TO reported-ZR_JMG_ORG_NODE.
      ENDIF.
    ENDLOOP.
  ENDMETHOD.

  METHOD get_global_authorizations.
    result-%create = COND #( WHEN requested_authorizations-%create = if_abap_behv=>mk-on
                              THEN if_abap_behv=>auth-allowed
                              ELSE if_abap_behv=>auth-unauthorized ).
  ENDMETHOD.

  METHOD get_instance_authorizations.
    result = VALUE #( FOR key IN keys
                       ( %tky    = key-%tky
                         %update = COND #( WHEN requested_authorizations-%update = if_abap_behv=>mk-on
                                            THEN if_abap_behv=>auth-allowed )
                         %delete = COND #( WHEN requested_authorizations-%delete = if_abap_behv=>mk-on
                                            THEN if_abap_behv=>auth-allowed ) ) ).
  ENDMETHOD.
ENDCLASS.

E-5 · ZBP_R_JMG_ORG_UPLD CLAS — 전역 + 로컬 핸들러

CLASS zbp_r_jmg_org_upld DEFINITION
  PUBLIC ABSTRACT FINAL FOR BEHAVIOR OF ZR_JMG_ORG_UPLD.
ENDCLASS.

CLASS zbp_r_jmg_org_upld IMPLEMENTATION.
ENDCLASS.

* --- Local Types (구현 인클루드) ---

CLASS lhc_ZR_JMG_ORG_UPLD DEFINITION INHERITING FROM cl_abap_behavior_handler.
  PRIVATE SECTION.
    TYPES: BEGIN OF ty_xlsx_row,
             node_id   TYPE zjmg_org_node_id,
             parent_id TYPE zjmg_org_node_id,
             org_name  TYPE zjmg_org_name,
             job_title TYPE zjmg_org_title,
           END OF ty_xlsx_row.

    METHODS get_global_authorizations FOR GLOBAL AUTHORIZATION
      IMPORTING REQUEST requested_authorizations FOR ZR_JMG_ORG_UPLD RESULT result.

    METHODS get_instance_authorizations FOR INSTANCE AUTHORIZATION
      IMPORTING keys REQUEST requested_authorizations FOR ZR_JMG_ORG_UPLD RESULT result.

    METHODS processUpload FOR MODIFY
      IMPORTING keys FOR ACTION ZR_JMG_ORG_UPLD~processUpload RESULT result.
ENDCLASS.

CLASS lhc_ZR_JMG_ORG_UPLD IMPLEMENTATION.
  METHOD get_global_authorizations.
    result-%create = COND #( WHEN requested_authorizations-%create = if_abap_behv=>mk-on
                              THEN if_abap_behv=>auth-allowed
                              ELSE if_abap_behv=>auth-unauthorized ).
  ENDMETHOD.

  METHOD get_instance_authorizations.
    result = VALUE #( FOR key IN keys
                       ( %tky    = key-%tky
                         %update = COND #( WHEN requested_authorizations-%update = if_abap_behv=>mk-on
                                            THEN if_abap_behv=>auth-allowed )
                         %delete = COND #( WHEN requested_authorizations-%delete = if_abap_behv=>mk-on
                                            THEN if_abap_behv=>auth-allowed ) ) ).
  ENDMETHOD.

  METHOD processUpload.
    READ ENTITIES OF ZR_JMG_ORG_UPLD IN LOCAL MODE
      ENTITY ZR_JMG_ORG_UPLD
        ALL FIELDS WITH CORRESPONDING #( keys )
      RESULT DATA(lt_upload).

    LOOP AT lt_upload INTO DATA(ls_upload).
      IF ls_upload-Attachment IS INITIAL.
        APPEND VALUE #( %tky = ls_upload-%tky ) TO failed-ZR_JMG_ORG_UPLD.
        APPEND VALUE #( %tky = ls_upload-%tky
                         %msg = NEW_MESSAGE_WITH_TEXT(
                                  text     = 'No file attached'
                                  severity = if_abap_behv_message=>severity-error ) )
               TO reported-ZR_JMG_ORG_UPLD.
        CONTINUE.
      ENDIF.

      DATA lt_rows TYPE STANDARD TABLE OF ty_xlsx_row WITH EMPTY KEY.
      CLEAR lt_rows.

      TRY.
          xco_cp_xlsx=>document->for_file_content( ls_upload-Attachment
            )->read_access( )->get_workbook(
            )->worksheet->at_position( 1
            )->select( xco_cp_xlsx_selection=>pattern_builder->simple_from_to(
                )->from_column( xco_cp_xlsx=>coordinate->for_alphabetic_value( 'A' )
                )->to_column( xco_cp_xlsx=>coordinate->for_alphabetic_value( 'D' )
                )->from_row( xco_cp_xlsx=>coordinate->for_numeric_value( 2 )
                )->get_pattern( )
            )->row_stream(
            )->operation->write_to( REF #( lt_rows )
            )->if_xco_xlsx_ra_operation~execute( ).
        CATCH cx_root INTO DATA(lx_xlsx).
          APPEND VALUE #( %tky = ls_upload-%tky ) TO failed-ZR_JMG_ORG_UPLD.
          APPEND VALUE #( %tky = ls_upload-%tky
                           %msg = NEW_MESSAGE_WITH_TEXT(
                                    text     = 'File could not be parsed as XLSX'
                                    severity = if_abap_behv_message=>severity-error ) )
                 TO reported-ZR_JMG_ORG_UPLD.
          CONTINUE.
      ENDTRY.

      " Trim stray whitespace from Excel cells (very common in real files):
      " IDs must not contain any blanks; texts keep internal single spaces.
      LOOP AT lt_rows ASSIGNING FIELD-SYMBOL(<ls_row>).
        CONDENSE <ls_row>-node_id NO-GAPS.
        CONDENSE <ls_row>-parent_id NO-GAPS.
        CONDENSE <ls_row>-org_name.
        CONDENSE <ls_row>-job_title.
      ENDLOOP.

      DELETE lt_rows WHERE node_id IS INITIAL.

      IF lt_rows IS INITIAL.
        APPEND VALUE #( %tky = ls_upload-%tky ) TO failed-ZR_JMG_ORG_UPLD.
        APPEND VALUE #( %tky = ls_upload-%tky
                         %msg = NEW_MESSAGE_WITH_TEXT(
                                  text     = 'No data rows found (data starts at row 2)'
                                  severity = if_abap_behv_message=>severity-error ) )
               TO reported-ZR_JMG_ORG_UPLD.
        CONTINUE.
      ENDIF.

      MODIFY ENTITIES OF ZR_JMG_ORG_NODE
        ENTITY ZR_JMG_ORG_NODE
          CREATE AUTO FILL CID
          FIELDS ( NodeID ParentID OrgName JobTitle )
          WITH VALUE #( FOR ls_row IN lt_rows
                         ( NodeID   = ls_row-node_id
                           ParentID = ls_row-parent_id
                           OrgName  = ls_row-org_name
                           JobTitle = ls_row-job_title ) )
        MAPPED DATA(ls_mapped)
        FAILED DATA(ls_failed)
        REPORTED DATA(ls_reported).

      IF ls_failed-zr_jmg_org_node IS NOT INITIAL.
        APPEND VALUE #( %tky = ls_upload-%tky ) TO failed-ZR_JMG_ORG_UPLD.
        APPEND VALUE #( %tky = ls_upload-%tky
                         %msg = NEW_MESSAGE_WITH_TEXT(
                                  text     = 'Some rows could not be created (check keys/duplicates)'
                                  severity = if_abap_behv_message=>severity-error ) )
               TO reported-ZR_JMG_ORG_UPLD.
        CONTINUE.
      ENDIF.

      APPEND VALUE #( %tky = ls_upload-%tky
                       %msg = NEW_MESSAGE_WITH_TEXT(
                                text     = |{ lines( lt_rows ) } nodes created from file|
                                severity = if_abap_behv_message=>severity-success ) )
             TO reported-ZR_JMG_ORG_UPLD.

      APPEND VALUE #( %tky   = ls_upload-%tky
                      %param = ls_upload ) TO result.
    ENDLOOP.
  ENDMETHOD.
ENDCLASS.
