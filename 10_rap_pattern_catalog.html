§1 데이터 조달 패턴 지도 기초

RAP/Fiori 앱의 데이터는 결국 어디서 계산되어 오는가로 갈립니다. 요구사항을 받았을 때 이 표에서 행 하나를 고르는 것이 설계의 첫 결정입니다.

① 일반 CDS 뷰DB (HANA)◎ 조인/union 자유최적 — 푸시다운✅기본값. 조인·계산으로 표현 가능한 모든 조회

§2 읽기(Query) 패턴 7종 기초심화

2.1 일반 CDS 뷰 기초

정의: define view entity ... as select from. 조인·association·CASE·집계로 선언하는 표준 패턴.
언제: 항상 여기서 시작합니다. 아래 ②~⑦은 전부 "①로 안 될 때"의 탈출구입니다.
코드: 09 Step 2의 YR_FLIGHT가 실물 예입니다. 문법 전체는 01_cds_concept.html.
한계: 선언형 SQL로 표현 불가한 로직(반복 계산, 절차적 분기), ABAP에만 있는 로직 재사용 불가.

2.2 파라미터 CDS 뷰 심화

정의: with parameters로 입력값을 받아 WHERE·계산에 쓰는 뷰. Open SQL에서 SELECT ... FROM zview( p_date = @lv_date )로 소비.

define view entity ZI_FLIGHT_ASOF
  with parameters p_date : s_date
  as select from ysflight
{
  key carrid, key connid, key fldate,
      price, currency
}
where fldate <= $parameters.p_date
함정(이 시스템 실측) — 파라미터 뷰를 Fiori Elements V4 List Report(UI5 1.96)로 직접 노출하면 $PARAMETER$ cannot be used for read 오류로 소비가 안 됩니다. 파라미터가 필요한 조회를 FE로 내보내야 하면 Custom Entity + Query Provider(§2.4)로 감싸서 파라미터를 필터(selectionField)로 받는 구조로 우회하세요 — §5의 제조원가가 정확히 이 이유로 그 구조입니다. (View Entity 파라미터 참조 문법은 $parameters.p_date — 콜론 아님, 01 자료 실측)

2.3 Table Function + AMDP — 실물: 수불부 심화

정의: CDS는 껍데기(파라미터+반환 구조)만 선언하고, 본체는 AMDP 클래스의 SQLScript(HANA 네이티브)가 구현. CDS인 척하지만 속은 DB 프로시저입니다.
언제: 선언형 CDS로 못 쓰는 로직 — 다단계 중간 결과 집합, 윈도 함수(ROW_NUMBER), 동적 폴백 규칙 등 — 을 DB 안에서 처리해야 할 때. 대량 데이터에서 앱서버 왕복 없이.

실물 — 자재원장 수불부 YTF_CO_SL_STOCK (선언부 발췌):

@EndUserText.label: 'CO-SL Stock Ledger Table Function'
define table function YTF_CO_SL_STOCK
  with parameters
    p_bwkey     : bwkey,
    p_gjahr     : gjahr,
    p_poper_from : poper,
    p_poper_to  : poper,
    p_curtp     : curtp,
    p_val_type  : char1,
    p_rldnr     : fins_ledger
returns
{
  client      : mandt;
  gjahr       : gjahr;
  ...
  opn_qty     : ml4h_lbkum;   -- 기초 수량/금액
  in_po_qty   : ml4h_lbkum;   -- 구매입고 등 입고 8분류
  out_sls_qty : ml4h_lbkum;   -- 판매출고 등 출고 10분류
  adj_ppv_amt : ml4h_salk3;   -- 조정 3분류
  cls_qty     : ml4h_lbkum;   -- 기말
  diff_qty    : ml4h_lbkum;   -- 검증용 차이(기초+입고-출고-기말)
}
implemented by method

구현 클래스 쪽 핵심 구조 (발췌 — 중간 결과 집합을 단계별로 쌓는 SQLScript 관용구):

METHOD get_stock_ledger BY DATABASE FUNCTION
                        FOR HDB LANGUAGE SQLSCRIPT
                        OPTIONS READ-ONLY
                        USING fcml_rep_v ckmlhd mara makt mldoc matdoc ytco_slmap.

    -- ① 마스터 브리지: 원가추정번호(kalnr) ↔ 자재 매핑
    lt_master = SELECT hd.mandt, hd.kalnr, hd.matnr, ...
        FROM ckmlhd AS hd ...
       WHERE hd.mandt = session_context('CLIENT')  -- AMDP는 자동 클라이언트 필터 없음!
         AND hd.bwkey = :p_bwkey;

    -- ② 기초/기말 잔액 (AB/EB 카테고리 피벗)
    lt_bal = SELECT ... SUM( CASE WHEN ml.categ = 'AB' ... ) ...;

    -- ③ 기중 이동 lt_doc → 자재문서 중복 제거 lt_mat → 이동 분류 lt_move
    -- ④ 매핑 테이블(ytco_slmap) 최소 seqnr 규칙 + ROW_NUMBER()로 1건 확정
    lt_ranked = SELECT ... ROW_NUMBER() OVER ( PARTITION BY src.rid ORDER BY map.seqnr ) AS rk ...
      WHERE rk = 1;

    -- ⑤ 카테고리 → 입고/출고/조정 컬럼 피벗(lt_pivot) → ⑥ 최종 RETURN SELECT
    RETURN SELECT ... diff_qty(기초+입고-출고-기말 검산 컬럼) ...;
ENDMETHOD.

이 로직은 ROW_NUMBER 파티션, 다단계 중간 집합, 와일드카드(*) 매핑 폴백을 씁니다 — 선언형 CDS 한 장으로는 불가능한 전형적 사례입니다. 전문(249줄)은 아래에.

YCL_CO_SL_AMDP=>GET_STOCK_LEDGER 전문 (실물 소스) 
CLASS ycl_co_sl_amdp DEFINITION
  PUBLIC FINAL CREATE PUBLIC.
  PUBLIC SECTION.
    INTERFACES if_amdp_marker_hdb.
    CLASS-METHODS get_stock_ledger FOR TABLE FUNCTION ytf_co_sl_stock.
ENDCLASS.

    METHOD get_stock_ledger BY DATABASE FUNCTION
                        FOR HDB LANGUAGE SQLSCRIPT
                        OPTIONS READ-ONLY
                        USING fcml_rep_v ckmlhd mara makt mldoc matdoc ytco_slmap.

    -- master bridge: kalnr <-> matnr/bwtar
    lt_master =
      SELECT hd.mandt, hd.kalnr, hd.matnr, hd.bwkey, hd.bwtar,
             makt.maktx, mara.mtart, mara.meins
        FROM ckmlhd AS hd
        INNER JOIN mara ON mara.mandt = hd.mandt AND mara.matnr = hd.matnr
        LEFT OUTER JOIN makt ON makt.mandt = hd.mandt AND makt.matnr = hd.matnr
             AND makt.spras = session_context('LOCALE_SAP')
       WHERE hd.mandt = session_context('CLIENT')
         AND hd.bwkey = :p_bwkey;

    -- master collapsed to material level for the FINAL join only:
    -- ckmlhd holds multiple kalnr per (matnr,bwkey,bwtar) (special-stock /
    -- sales-order segments) -> joining lt_master directly fans out result rows
    -- (verified: TG14 x10 duplicates). Aggregation tables (bal/move) still use
    -- lt_master at kalnr level, which is correct.
    lt_mast1 =
      SELECT mandt, matnr, bwkey, bwtar,
             MAX(maktx) AS maktx, MAX(mtart) AS mtart, MAX(meins) AS meins
        FROM :lt_master
       GROUP BY mandt, matnr, bwkey, bwtar;

    -- opening (AB @from) / closing (EB @to). rldnr filtered to avoid parallel-ledger double count.
    lt_bal =
      SELECT ml.mandt, mas.matnr, mas.bwkey, mas.bwtar, ml.waers,
        SUM( CASE WHEN ml.categ = 'AB' AND ml.poper = :p_poper_from
                  THEN ml.lbkum ELSE 0 END ) AS opn_qty,
        SUM( CASE WHEN ml.categ = 'AB' AND ml.poper = :p_poper_from THEN
               CASE WHEN :p_val_type = 'A'
                    THEN ml.salk3 + ml.estprd + ml.estkdm + ml.mstprd + ml.mstkdm
                    ELSE ml.salk3 END
             ELSE 0 END ) AS opn_amt,
        SUM( CASE WHEN ml.categ = 'EB' AND ml.poper = :p_poper_to
                  THEN ml.lbkum ELSE 0 END ) AS cls_qty,
        SUM( CASE WHEN ml.categ = 'EB' AND ml.poper = :p_poper_to THEN
               CASE WHEN :p_val_type = 'A'
                    THEN ml.salk3 + ml.estprd + ml.estkdm + ml.mstprd + ml.mstkdm
                    ELSE ml.salk3 END
             ELSE 0 END ) AS cls_amt
      FROM fcml_rep_v AS ml
      INNER JOIN :lt_master AS mas
        ON mas.mandt = ml.mandt AND mas.kalnr = ml.kalnr_mat
      WHERE ml.mandt = session_context('CLIENT')
        AND ml.bdatj = :p_gjahr
        AND ml.curtp = :p_curtp
        AND ml.rldnr = :p_rldnr
        AND ml.categ IN ('AB','EB')
        AND ml.poper IN (:p_poper_from, :p_poper_to)
      GROUP BY ml.mandt, mas.matnr, mas.bwkey, mas.bwtar, ml.waers;

    -- movement source lines FIRST (filter early), so the matdoc lookup below
    -- can be restricted to referenced documents instead of a full-table scan.
    lt_doc =
      SELECT doc.mandt, mas.matnr, mas.bwtar, mas.bwkey, doc.waers,
             doc.categ, doc.ptyp, ifnull(doc.glvor,'') AS glvor,
             ifnull(doc.runref,'') AS runref,
             doc.awtyp, doc.awref, doc.aworg, doc.awitem,
             doc.quant AS move_qty,
             CASE WHEN :p_val_type = 'A'
                  THEN doc.stval + doc.prd + doc.kdm
                  ELSE doc.stval END AS move_amt
      FROM mldoc AS doc
      INNER JOIN :lt_master AS mas
        ON mas.mandt = doc.mandt AND mas.kalnr = doc.kalnr
      WHERE doc.mandt = session_context('CLIENT')
        AND doc.curtp = :p_curtp
        AND doc.rldnr = :p_rldnr
        AND doc.jahrper BETWEEN :p_gjahr || :p_poper_from
                            AND :p_gjahr || :p_poper_to
        AND doc.categ IN ('ZU','VN','PC');

    -- matdoc deduplicated to one row per (mblnr,mjahr,zeile): physical key is a GUID,
    -- so (mblnr,mjahr,zeile) is NOT unique -> a direct join fans out and double counts.
    -- MAX() is deterministic since split rows of a line share bwart/shkzg.
    -- Restricted to documents actually referenced by lt_doc (was: full matdoc scan).
    lt_mat =
      SELECT mat.mandt, mat.mblnr, mat.mjahr, mat.zeile,
             MAX(mat.bwart) AS bwart, MAX(mat.shkzg) AS shkzg, MAX(mat.sobkz) AS sobkz
        FROM matdoc AS mat
        INNER JOIN ( SELECT DISTINCT mandt, awref, aworg, awitem
                       FROM :lt_doc
                      WHERE awtyp = 'MKPF' ) AS d
          ON  d.mandt = mat.mandt
          AND d.awref = mat.mblnr
          AND CAST( mat.mjahr AS INTEGER ) = CAST( d.aworg  AS INTEGER )
          AND CAST( mat.zeile AS INTEGER ) = CAST( d.awitem AS INTEGER )
       GROUP BY mat.mandt, mat.mblnr, mat.mjahr, mat.zeile;

    -- movement lines: value=quant/stval, coarse class=categ/glvor/ptyp/runref (mldoc)
    lt_move =
      SELECT
        ROW_NUMBER() OVER () AS rid,
        src.mandt, src.matnr, src.bwtar, src.bwkey, src.waers,
        src.categ, src.ptyp, src.glvor, src.runref,
        ifnull(mat.bwart,'') AS bwart,
        ifnull(mat.shkzg,'') AS shkzg,
        ifnull(mat.sobkz,'') AS sobkz,
        src.move_qty, src.move_amt
      FROM :lt_doc AS src
      LEFT OUTER JOIN :lt_mat AS mat
        ON  src.awtyp = 'MKPF'
        AND mat.mandt = src.mandt
        AND mat.mblnr = src.awref
        AND CAST( mat.mjahr AS INTEGER ) =
            CASE WHEN src.awtyp = 'MKPF' THEN CAST( src.aworg AS INTEGER ) ELSE -1 END
        AND CAST( mat.zeile AS INTEGER ) =
            CASE WHEN src.awtyp = 'MKPF' THEN CAST( src.awitem AS INTEGER ) ELSE -1 END;

    -- mapping: min-seqnr rule where each dim matches OR '*'. Unmatched falls back by categ;
    -- PC split by runref: costing-run settlement (runref<>'') -> ADJ_ML, else ADJ_REV.
    lt_ranked =
      SELECT rid, mandt, matnr, bwtar, bwkey, waers, move_qty, move_amt, cat_code
      FROM (
        SELECT src.rid, src.mandt, src.matnr, src.bwtar, src.bwkey, src.waers,
               src.move_qty, src.move_amt,
               ifnull( map.cat_code,
                       CASE WHEN src.categ = 'ZU' THEN 'IN_OTH'
                            WHEN src.categ = 'VN' THEN 'OUT_OTH'
                            WHEN src.categ = 'PC' AND src.runref <> '' THEN 'ADJ_ML'
                            WHEN src.categ = 'PC' THEN 'ADJ_REV'
                            ELSE 'ADJ_REV' END ) AS cat_code,
               ROW_NUMBER() OVER ( PARTITION BY src.rid ORDER BY map.seqnr ASC ) AS rk
        FROM :lt_move AS src
        LEFT OUTER JOIN ytco_slmap AS map
          ON  map.mandt = src.mandt
          AND ( map.categ = src.categ OR map.categ = '*' )
          AND ( map.ptyp  = src.ptyp  OR map.ptyp  = '*' )
          AND ( map.glvor = src.glvor OR map.glvor = '*' )
          AND ( map.bwart = src.bwart OR map.bwart = '*' )
          AND ( map.shkzg = src.shkzg OR map.shkzg = '*' )
          AND ( map.sobkz = src.sobkz OR map.sobkz = '*' )
      )
      WHERE rk = 1;

    -- pivot: inflow=original sign, outflow=x(-1) positive, adjustment=original sign. adj qty=0.
    lt_pivot =
      SELECT mandt, matnr, bwtar, bwkey, MAX(waers) AS waers,
        SUM( CASE WHEN cat_code='IN_PO'  THEN move_qty ELSE 0 END ) AS in_po_qty,
        SUM( CASE WHEN cat_code='IN_PO'  THEN move_amt ELSE 0 END ) AS in_po_amt,
        SUM( CASE WHEN cat_code='IN_SUB' THEN move_qty ELSE 0 END ) AS in_sub_qty,
        SUM( CASE WHEN cat_code='IN_SUB' THEN move_amt ELSE 0 END ) AS in_sub_amt,
        SUM( CASE WHEN cat_code='IN_PRD' THEN move_qty ELSE 0 END ) AS in_prd_qty,
        SUM( CASE WHEN cat_code='IN_PRD' THEN move_amt ELSE 0 END ) AS in_prd_amt,
        SUM( CASE WHEN cat_code='IN_TRF' THEN move_qty ELSE 0 END ) AS in_trf_qty,
        SUM( CASE WHEN cat_code='IN_TRF' THEN move_amt ELSE 0 END ) AS in_trf_amt,
        SUM( CASE WHEN cat_code='IN_RET' THEN move_qty ELSE 0 END ) AS in_ret_qty,
        SUM( CASE WHEN cat_code='IN_RET' THEN move_amt ELSE 0 END ) AS in_ret_amt,
        SUM( CASE WHEN cat_code='IN_MTM' THEN move_qty ELSE 0 END ) AS in_mtm_qty,
        SUM( CASE WHEN cat_code='IN_MTM' THEN move_amt ELSE 0 END ) AS in_mtm_amt,
        SUM( CASE WHEN cat_code='IN_ADJ' THEN move_qty ELSE 0 END ) AS in_adj_qty,
        SUM( CASE WHEN cat_code='IN_ADJ' THEN move_amt ELSE 0 END ) AS in_adj_amt,
        SUM( CASE WHEN cat_code='IN_OTH' THEN move_qty ELSE 0 END ) AS in_oth_qty,
        SUM( CASE WHEN cat_code='IN_OTH' THEN move_amt ELSE 0 END ) AS in_oth_amt,
        SUM( CASE WHEN cat_code='OUT_SLS' THEN -move_qty ELSE 0 END ) AS out_sls_qty,
        SUM( CASE WHEN cat_code='OUT_SLS' THEN -move_amt ELSE 0 END ) AS out_sls_amt,
        SUM( CASE WHEN cat_code='OUT_PRD' THEN -move_qty ELSE 0 END ) AS out_prd_qty,
        SUM( CASE WHEN cat_code='OUT_PRD' THEN -move_amt ELSE 0 END ) AS out_prd_amt,
        SUM( CASE WHEN cat_code='OUT_SUB' THEN -move_qty ELSE 0 END ) AS out_sub_qty,
        SUM( CASE WHEN cat_code='OUT_SUB' THEN -move_amt ELSE 0 END ) AS out_sub_amt,
        SUM( CASE WHEN cat_code='OUT_TRF' THEN -move_qty ELSE 0 END ) AS out_trf_qty,
        SUM( CASE WHEN cat_code='OUT_TRF' THEN -move_amt ELSE 0 END ) AS out_trf_amt,
        SUM( CASE WHEN cat_code='OUT_RTV' THEN -move_qty ELSE 0 END ) AS out_rtv_qty,
        SUM( CASE WHEN cat_code='OUT_RTV' THEN -move_amt ELSE 0 END ) AS out_rtv_amt,
        SUM( CASE WHEN cat_code='OUT_CC'  THEN -move_qty ELSE 0 END ) AS out_cc_qty,
        SUM( CASE WHEN cat_code='OUT_CC'  THEN -move_amt ELSE 0 END ) AS out_cc_amt,
        SUM( CASE WHEN cat_code='OUT_EXP' THEN -move_qty ELSE 0 END ) AS out_exp_qty,
        SUM( CASE WHEN cat_code='OUT_EXP' THEN -move_amt ELSE 0 END ) AS out_exp_amt,
        SUM( CASE WHEN cat_code='OUT_MTM' THEN -move_qty ELSE 0 END ) AS out_mtm_qty,
        SUM( CASE WHEN cat_code='OUT_MTM' THEN -move_amt ELSE 0 END ) AS out_mtm_amt,
        SUM( CASE WHEN cat_code='OUT_ADJ' THEN -move_qty ELSE 0 END ) AS out_adj_qty,
        SUM( CASE WHEN cat_code='OUT_ADJ' THEN -move_amt ELSE 0 END ) AS out_adj_amt,
        SUM( CASE WHEN cat_code='OUT_OTH' THEN -move_qty ELSE 0 END ) AS out_oth_qty,
        SUM( CASE WHEN cat_code='OUT_OTH' THEN -move_amt ELSE 0 END ) AS out_oth_amt,
        SUM( CASE WHEN cat_code='ADJ_PPV' THEN move_amt ELSE 0 END ) AS adj_ppv_amt,
        SUM( CASE WHEN cat_code='ADJ_REV' THEN move_amt ELSE 0 END ) AS adj_rev_amt,
        SUM( CASE WHEN cat_code='ADJ_ML'  THEN move_amt ELSE 0 END ) AS adj_ml_amt
      FROM :lt_ranked

Query Provider 구현 — 요청 해석 → 엔진 위임 → 응답 채우기의 3막 구조를 해부합니다:

CLASS ycl_co_cgm_qry DEFINITION PUBLIC FINAL CREATE PUBLIC.
  PUBLIC SECTION.
    INTERFACES if_rap_query_provider.
ENDCLASS.

CLASS ycl_co_cgm_qry IMPLEMENTATION.
  METHOD if_rap_query_provider~select.
    DATA lt_out TYPE STANDARD TABLE OF yc_cgm_stmt WITH DEFAULT KEY.
    TRY.
        " ─ 1막: 요청 해석 — FE 필터바의 조건이 ranges로 들어온다
        DATA(lt_ranges) = io_request->get_filter( )->get_as_ranges( ).
        LOOP AT lt_ranges REFERENCE INTO DATA(lr_pair).
          ASSIGN lr_pair->range TO FIELD-SYMBOL(<ranges>).
          IF <ranges> IS NOT ASSIGNED OR <ranges> IS INITIAL. CONTINUE. ENDIF.
          ASSIGN <ranges>[ 1 ] TO FIELD-SYMBOL(<row>).
          IF <row> IS NOT ASSIGNED. CONTINUE. ENDIF.
          ASSIGN COMPONENT 'LOW' OF STRUCTURE <row> TO FIELD-SYMBOL(<low>).
          IF <low> IS NOT ASSIGNED. CONTINUE. ENDIF.
          CASE to_upper( lr_pair->name ).
            WHEN 'CONTROLLINGAREA'. lv_kokrs = <low>.
            WHEN 'COMPANYCODE'.     lv_bukrs = <low>.
            WHEN 'FISCALYEAR'.      lv_gjahr = <low>.
            WHEN 'PERIOD'.          lv_poper = <low>.
            WHEN 'LEDGER'.          lv_rldnr = <low>.
          ENDCASE.
        ENDLOOP.
        IF lv_poper IS INITIAL. lv_poper = sy-datum+4(2). ENDIF.   " 기본값 처리
        IF lv_rldnr IS INITIAL. lv_rldnr = '0L'. ENDIF.

        " ─ 2막: 위임 — 화면 무관하게 재사용되는 엔진 클래스 호출
        IF lv_kokrs IS NOT INITIAL AND lv_bukrs IS NOT INITIAL AND lv_gjahr IS NOT INITIAL.
          DATA(lt_rows) = NEW ycl_co_cgm( )->get_statement_compare(
            iv_kokrs = lv_kokrs iv_bukrs = lv_bukrs iv_gjahr = lv_gjahr
            iv_poper = lv_poper iv_rldnr = lv_rldnr ).
          LOOP AT lt_rows INTO DATA(ls).
            APPEND VALUE #( controllingarea = lv_kokrs companycode = lv_bukrs
              fiscalyear = lv_gjahr period = lv_poper ledger = lv_rldnr
              displayseq = ls-display_seq linecode = ls-line_code sectioncode = ls-sect
              linetext = ls-line_text issection = ls-is_sect
              curamount = ls-cur_amt curtotal = ls-cur_tot
              priamount = ls-pri_amt pritotal = ls-pri_tot currency = ls-waers ) TO lt_out.
          ENDLOOP.
        ENDIF.
      CATCH cx_root.
        CLEAR lt_out.
    ENDTRY.

    " ─ 3막: 응답 — 건수 요청 시 total 설정 + 데이터 전달
    IF io_request->is_total_numb_of_rec_requested( ).
      io_response->set_total_number_of_records( lines( lt_out ) ).
    ENDIF.
    io_response->set_data( lt_out ).
  ENDMETHOD.
ENDCLASS.

  • 페이징·정렬은 개발자 책임 — 일반 CDS라면 프레임워크가 하는 $top/$skip/정렬 처리를 여기선 직접 해야 합니다(이 실물은 결과가 명세서 수십 행이라 전량 반환 + presentationVariant 정렬로 충분). 대량 데이터면 io_request->get_paging( )·get_sort_elements( )를 반드시 처리하세요.
  • 한계 — CDS 합성 불가(다른 뷰가 이 엔티티를 SELECT 못 함), Open SQL 소비 불가. 오직 OData/서비스 경유.

CLASS lhc_flight IMPLEMENTATION.

  METHOD get_global_authorizations.
  ENDMETHOD.

  METHOD create.
    LOOP AT entities INTO DATA(ls_ent).
      " 즉시 검증(레거시 검증 함수의 check-only 모드 재사용)
      CALL FUNCTION 'Z_FLIGHT_MAINTAIN'
        EXPORTING iv_check_only = abap_true
                  is_flight     = CORRESPONDING ysflight( ls_ent MAPPING FROM ENTITY )
        EXCEPTIONS invalid_data = 1 OTHERS = 2.
      IF sy-subrc <> 0.
        APPEND VALUE #( %cid = ls_ent-%cid ) TO failed-flight.
        APPEND VALUE #( %cid = ls_ent-%cid
                        %msg = new_message_from_symsg( ) ) TO reported-flight.
        CONTINUE.
      ENDIF.
      " 통과 → 버퍼에만 기록 (DB 접근 없음!)
      APPEND VALUE #( instance = ls_ent flag = 'C' ) TO gt_buffer.
      " 프레임워크에 키 응답 (early numbering 가정 시)
      APPEND VALUE #( %cid = ls_ent-%cid
                      %key = ls_ent-%key ) TO mapped-flight.
    ENDLOOP.
  ENDMETHOD.

  METHOD update.
    LOOP AT entities INTO DATA(ls_ent).
      " 현재 상태 + 변경분 병합 후 버퍼에 'U'로 기록 (병합 로직 생략 표기)
      APPEND VALUE #( instance = CORRESPONDING #( ls_ent ) flag = 'U' ) TO gt_buffer.
    ENDLOOP.
  ENDMETHOD.

  METHOD delete.
    LOOP AT keys INTO DATA(ls_key).
      APPEND VALUE #( instance = CORRESPONDING #( ls_key ) flag = 'D' ) TO gt_buffer.
    ENDLOOP.
  ENDMETHOD.

  METHOD read.
    " 버퍼 우선, 없으면 DB — unmanaged에선 read도 내 책임
    LOOP AT keys INTO DATA(ls_key).
      SELECT SINGLE * FROM ysflight
        WHERE carrid = @ls_key-CarrierId
          AND connid = @ls_key-ConnectionId
          AND fldate = @ls_key-FlightDate
        INTO @DATA(ls_db).
      IF sy-subrc = 0.
        APPEND CORRESPONDING #( ls_db MAPPING TO ENTITY ) TO result.
      ELSE.
        APPEND VALUE #( %tky = ls_key-%tky ) TO failed-flight.
      ENDIF.
    ENDLOOP.
  ENDMETHOD.

  METHOD lock.
    " 레거시 잠금 오브젝트 재사용 — RAP이 요청 처리 동안 호출
    LOOP AT keys INTO DATA(ls_key).
      CALL FUNCTION 'ENQUEUE_EZFLIGHT'
        EXPORTING carrid         = ls_key-CarrierId
                  connid         = ls_key-ConnectionId
                  fldate         = ls_key-FlightDate
        EXCEPTIONS foreign_lock  = 1 system_failure = 2 OTHERS = 3.
      IF sy-subrc <> 0.
        APPEND VALUE #( %tky = ls_key-%tky ) TO failed-flight.
        APPEND VALUE #( %tky = ls_key-%tky
                        %msg = new_message_from_symsg( ) ) TO reported-flight.
      ENDIF.
    ENDLOOP.
  ENDMETHOD.

ENDCLASS.

§6 실무 함정·노하우 모음 심화

전부 이 시스템(S/4HANA 2021 / 756 / UI5 1.96)에서 실제로 부딪혀 확인한 것들입니다. 증상 → 원인 → 해법 3줄 형식.

1. strict 숫자형 구문오류

증상: 온라인 예제의 strict ( 2 );를 BDEF에 넣으면 "Unexpected character".
원인: 숫자형 strict 모드는 2022(7.57)+. 756은 무인자 strict;까지만.
해법: strict;로 교체. (클라우드/최신 예제 이식 시 1순위 점검 항목)

2. SRVB 바인딩 버전은 사후 변경 불가

증상: V2로 만든 SRVB를 V4로 바꾸고 싶은데 편집 불가. 이름만 "O4"인 V2 바인딩으로 FE 프리뷰가 깨진 실사례 있음.
원인: 바인딩 타입/버전은 생성 시 고정되는 속성.
해법: (게시했다면 unpublish 후) 삭제 → 올바른 버전으로 재생성. 확인은 DB로: SELECT srvb_name, bind_type_version FROM srvb_bind_type WHERE srvb_name = '...'.

3. V4 서비스 Publish가 ADT에서 실패

증상: SRVB Publish 시 /IWFND/CM_DESIGNTIME 174 "Transport request not available and client is set to auto-record on".
원인: 클라이언트가 변경 자동기록 설정 → 게시가 커스터마이징 요청을 요구하는데 ADT(HTTP)는 요청 선택 팝업을 못 띄움.
해법: SAP GUI /IWFND/V4_ADMIN에서 게시(팝업 정상 동작). 또는 SCC4에서 해당 클라이언트 자동기록 해제(운영 정책 협의 필요).

4. @UI.facet 인라인 활성화 오류

증상: CDS 뷰에 @UI.facet을 인라인으로 넣었을 때 활성화 오류가 나는 경우가 있음(뷰 종류·위치 의존 — 이번 실물 YC_FLIGHT에선 인라인 동작 확인).
원인: facet은 뷰 레벨 어노테이션 배열이라 배치 제약이 까다로움.
해법: 오류가 나면 Metadata Extension(DDLX)으로 분리가 정석 — 뷰에 @Metadata.allowExtensions: true를 달고 DDLX에 @UI 전체 이관. UI 변경과 데이터 모델 변경의 전송(CTS) 분리라는 부수 이득도 큼.

5. 파라미터 CDS 뷰의 FE V4 소비 불가

증상: 파라미터 뷰를 V4 List Report(UI5 1.96)로 노출하면 $PARAMETER$ cannot be used for read.
원인: 이 UI5 버전의 FE V4가 파라미터화된 엔티티 read 미지원.
해법: Custom Entity + Query Provider로 감싸 파라미터를 selectionField 필터로 수신(§2.4 실물이 그 사례).

6. CDS Hierarchy는 되지만 트리 화면이 안 나온다

증상: define hierarchy 활성·조회 정상인데 FE 화면에 트리가 없음.
원인: UI5 1.96에 FE V4 recursiveHierarchy(트리 테이블) 렌더러 부재 — 1.120+ GA.
해법: 이 시스템에선 트리 UI는 GUI(cl_gui_alv_tree)로, RAP 트리는 최신 UI5 환경(BTP 등)에서. "모델 가능 ≠ 화면 이득"을 항상 UI5 버전과 함께 검토.

7. draft 테이블은 손대지 말 것

증상: draft 테이블을 수작업 수정/재생성하다 BO 활성화가 꼬임.
원인: draft 테이블 구조는 BDEF·CDS와 정합해야 하는 파생물.
해법: 항상 ADT 퀵픽스(BDEF의 draft table에서 Ctrl+1)로 생성·갱신. 필드 추가 시 순서: CDS 수정 → 퀵픽스로 draft 테이블 재생성 → 활성화.

8. 활성화 순서 — BDEF 먼저

증상: RAP 오브젝트 무더기 활성화 시 상호참조 오류.
원인: BDEF와 behavior pool, interface와 projection이 서로 참조.
해법: 순서는 테이블 → CDS(interface→projection) → BDEF → 클래스 → SRVD → SRVB. ADT에서 여럿 선택해 일괄 활성화(Ctrl+Shift+F3)가 가장 안전.
