1. 개요 — 항공편 데이터 모델이란

SAP가 교육·데모용으로 제공하는 항공편 예약 데이터 모델(SAP Flight / BC_TRAVEL)입니다. 항공사·운항 스케줄·개별 항공편·예약·승객·여행사·공항·기종이 서로 외래키로 연결된, 실무 데이터 모델을 축소한 완결된 예제입니다. 전 세계 ABAP/CDS 교재의 공통 언어라, 이 모델만 익히면 다른 자료와 바로 교차 학습할 수 있습니다.

핵심 축은 항공사 → 운항 → 항공편 → 예약 4단계이고, 여기에 공항·기종·승객·여행사 마스터가 붙습니다.

클라이언트 주의 — 모든 테이블 첫 필드는 MANDT(클라이언트)입니다. 조회·조인 시 항상 클라이언트를 함께 걸어야 합니다(ABAP은 자동, AMDP는 수동 → 03 AMDP 5.2).

2. ERD — 한눈에 보는 관계

핵심 축(항공사→운항→항공편→예약) 마스터 테이블 1 : N 1대다 관계 (부모:자식)

화살표(선)의 1쪽이 부모, N쪽이 자식입니다. 예: 한 항공사(SCARR)에 여러 운항(SPFLI)이 딸리고, 한 운항에 여러 날짜의 항공편(SFLIGHT)이, 한 항공편에 여러 예약(SBOOK)이 딸립니다. 예약은 다시 고객(SCUSTOM)·여행사(STRAVELAG)를 참조합니다.

6. 실제 데이터로 보는 관계

실습 시스템의 예약 한 건(AA / 0017 / 2023-06-22 / BOOKID 00000061)을 모든 관련 테이블로 실제 추적한 결과입니다. 아래 값은 서버에서 조인해 뽑은 실측 데이터입니다.

  • SBOOK — BOOKID 00000061 · CLASS Y · 결제 437.95 EUR / 항공사통화 401.79 USD · CUSTOMID 00001363 · AGENCYNUM 00000284
    ↓ CUSTOMID · AGENCYNUM · (CARRID+CONNID+FLDATE)로 연결
  • SCUSTOM (ID 00001363) — 승객 Lothar Koller
  • STRAVELAG (AGENCYNUM 00000284) — 여행사 Rainy, Stormy, Cloudy
  • SFLIGHT (AA/0017/2023-06-22) — 기종 747-400 · 가격 423.44 USD
    ↓ PLANETYPE · (CARRID+CONNID)로 연결
  • SAPLANE (747-400) — Boeing 광동체기 (좌석/제원)
  • SPFLI (AA/0017) — 노선 NEW YORK (JFK) → SAN FRANCISCO (SFO)
    ↓ AIRPFROM=JFK · AIRPTO=SFO · CARRID로 연결
  • SAIRPORT — JFK / SFO (공항 마스터)
  • SCARR (AA) — American Airlines (USD)
읽는 법 — 예약(SBOOK) 한 줄에서 시작해 키를 따라가면 "누가(SCUSTOM) 어느 여행사(STRAVELAG)를 통해 어떤 항공사(SCARR)의 어느 노선(SPFLI)·어느 날짜 편(SFLIGHT)을 어떤 기종(SAPLANE)으로, 어느 공항(SAIRPORT) 사이를 예약했는가"가 모두 복원됩니다. 이것이 정규화된 관계형 모델의 힘입니다.

7. 조인 쿼리 예시

7.1 ABAP Open SQL — 예약 1건을 전 테이블로 추적

SELECT SINGLE
    b~bookid, sc~carrname, sp~cityfrom, sp~airpfrom, sp~cityto, sp~airpto,
    f~planetype, f~price, f~currency, cu~name AS passenger, ag~name AS agency
  FROM sbook AS b
  INNER JOIN scarr    AS sc ON sc~carrid = b~carrid
  INNER JOIN spfli    AS sp ON sp~carrid = b~carrid AND sp~connid = b~connid
  INNER JOIN sflight  AS f  ON f~carrid  = b~carrid AND f~connid  = b~connid
                                    AND f~fldate  = b~fldate
  INNER JOIN scustom  AS cu ON cu~id = b~customid
  LEFT OUTER JOIN stravelag AS ag ON ag~agencynum = b~agencynum
  WHERE b~carrid = 'AA' AND b~connid = '0017'
    AND b~fldate = '20230622' AND b~bookid = '00000061'
  INTO @DATA(ls).

여행사는 LEFT OUTER JOIN: AGENCYNUM이 0(직판)인 예약도 유지하기 위해서입니다.

7.2 CDS View — association으로 표현

define view entity ZI_BOOKING_DETAIL as select from sbook
  association [1..1] to scarr   as _Carrier  on $projection.CarrierID = _Carrier.carrid
  association [1..1] to scustom as _Customer on $projection.CustomerID = _Customer.id
{
  key carrid as CarrierID, key connid as ConnectionID,
  key fldate as FlightDate, key bookid as BookingID,
      customid as CustomerID,
      _Carrier.carrname  as CarrierName,
      _Customer.name     as PassengerName
}

관계를 association으로 선언하면 경로식(_Carrier.carrname)으로 자연스럽게 참조합니다 → CDS 3부.

7.3 집계 — 항공사·노선별 예약/매출

SELECT b~carrid, b~connid, sc~carrname,
       COUNT(*) AS bookings, SUM( b~loccuram ) AS revenue
  FROM sbook AS b
  INNER JOIN scarr AS sc ON sc~carrid = b~carrid
  WHERE b~cancelled = @space
  GROUP BY b~carrid, b~connid, sc~carrname
  ORDER BY revenue DESCENDING
  INTO TABLE @DATA(lt).
